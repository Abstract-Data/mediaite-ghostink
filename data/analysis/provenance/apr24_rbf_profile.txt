Interpretation (Phase-15 F0):
  PELT's RBF cost model materializes an O(n²) pairwise kernel matrix for
  each feature column per author. On isaac-schorr's ~1,050 articles the
  kernel matrix is ~550k entries; across 23 features the per-author PELT
  phase issues >3M individual ``costrbf.error`` calls. This dominates
  wall-clock by a factor of ~100× over every other analysis step combined.

  L2 cost (``ruptures.Pelt(model="l2")``) implements direct-formula
  segment-sum bookkeeping: O(n) per feature, no pairwise matrix. Expected
  throughput is 100–500× on the PELT phase alone. Phase F0.1 in the
  implementation plan is the one-line knob flip that delivers this
  improvement; every other Phase-15 performance change is a single-digit
  percent improvement sitting on top of F0.
