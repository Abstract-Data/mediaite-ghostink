## Objective

Build the reporting pipeline: create the 7 Jupyter notebooks that form the analysis narrative, configure Quarto for rendering, and set up Cloudflare Pages deployment. After this phase, `uv run forensics report` generates the full report and `make deploy` publishes it.

## 1. Notebook Structure

Create 7 notebooks in `notebooks/` that import from `src/forensics/` for logic and contain narrative, visualization, and high-level calls:

### notebooks/01_scraping.ipynb

**Purpose:** Document the scraping process, show article counts and date ranges per author.

Cells:
1. **Intro markdown:** Describe the data collection methodology (two-step scraping)
2. **Load config:** Import settings, connect to SQLite
3. **Author summary table:** For each configured author, show name, role, total articles, date range (earliest to latest)
4. **Article volume over time:** Plotly bar chart of articles per month per author
5. **Data quality summary:** Count of redirects, duplicates, extraction failures
6. **Corpus statistics:** Total articles, total words, mean/median article length

### notebooks/02_exploration.ipynb

**Purpose:** Initial data exploration, corpus statistics, date distributions.

Cells:
1. **Word count distribution:** Histogram of article lengths per author (Plotly)
2. **Publication cadence:** Articles per week heatmap, year × week (Plotly heatmap)
3. **Article length over time:** Scatter plot with rolling average trend line
4. **Day-of-week distribution:** Bar chart showing publication patterns
5. **Missing data audit:** Any gaps in coverage, months with zero articles

### notebooks/03_feature_analysis.ipynb

**Purpose:** Feature extraction results, per-feature time series, baseline characterization.

Cells:
1. **Feature correlation matrix:** Heatmap of Pearson correlations between all numeric features
2. **Baseline characterization:** For each author's baseline period, compute mean ± std for every feature. Present as a styled table.
3. **Per-feature time series:** For the top 10 most discriminative features, show:
   - Raw values as scatter plot
   - 30-day and 90-day rolling averages with confidence bands
   - Annotated with baseline period shading
4. **Feature distribution comparison:** Violin plots comparing baseline vs. recent periods
5. **AI marker frequency timeline:** Dedicated chart for AI marker phrase frequency over time

### notebooks/04_change_point_detection.ipynb

**Purpose:** PELT + BOCPD results, cross-feature convergence.

Cells:
1. **Change point summary table:** Feature, date, method, confidence, effect size, direction
2. **Feature heatmap over time:** Rows = features, columns = months, color = z-score relative to baseline. Change points marked with vertical lines.
3. **Convergence windows:** Highlight time periods where 60%+ of features agree on a change. Show a stacked chart of which features triggered.
4. **BOCPD probability plot:** For selected features, show the posterior probability of a change point over time (the gradual shift visualization)
5. **Effect size summary:** Forest plot of Cohen's d for all significant change points

### notebooks/05_embedding_drift.ipynb

**Purpose:** Centroid tracking, UMAP visualization, AI baseline comparison.

Cells:
1. **UMAP trajectory plot:** 2D projection of monthly centroids, color-coded by year. Target author trajectory vs. control author trajectories. AI baseline centroid marked with a distinct symbol.
2. **Cosine similarity decay curve:** Author's similarity to their own baseline over time. Annotated with convergence windows from Phase 7.
3. **Centroid velocity timeline:** Monthly centroid movement speed, with spike annotations.
4. **AI convergence chart:** Similarity to AI baseline centroid over time (if AI baseline exists). Dual-axis with baseline similarity decay for comparison.
5. **Intra-period variance trend:** Within-month embedding variance over time.
6. **Control comparison overlay:** Same metrics for control authors on the same charts.

### notebooks/06_statistical_evidence.ipynb

**Purpose:** Hypothesis test results, effect sizes, control comparison, convergence scoring.

Cells:
1. **Hypothesis test results table:** All tests with raw p-value, corrected p-value, effect size, CI, significant (yes/no). Styled with significant rows highlighted.
2. **Multiple comparison visualization:** Show the Benjamini-Hochberg step-up procedure graphically (ranked p-values vs. BH threshold line)
3. **Effect size forest plot:** All significant features with Cohen's d and 95% CI
4. **Control comparison dashboard:** Side-by-side metrics for target vs. each control:
   - Number of significant change points
   - Mean effect size
   - Convergence window count
   - Drift score
5. **Editorial vs. Author attribution:** Bar chart of signal attribution score
6. **Convergence score summary:** Pipeline A score vs. Pipeline B score for each convergence window

### notebooks/07_full_report.ipynb

**Purpose:** Assembled final report with executive summary, all key charts, methodology appendix.

Sections (as markdown cells + key charts pulled from earlier notebooks):

1. **Executive Summary**
   - Headline finding (1-2 sentences)
   - Article count, time span
   - Number of significant change points and convergence windows
   - Statistical significance summary
   - Key chart: single most compelling visualization

2. **Author Profile**
   - Baseline voice characterization from earliest stable period
   - Summary statistics table for baseline features

3. **Productivity Analysis**
   - Publication frequency, word count trends
   - Burst detection results
   - Output volume timeline

4. **Stylometric Timeline**
   - Feature heatmap (from notebook 04)
   - Top 5 most significant feature time series with change points marked
   - Rolling averages with confidence bands

5. **Embedding Drift Analysis**
   - UMAP trajectory (from notebook 05)
   - Cosine similarity decay curve
   - AI convergence (if applicable)

6. **Statistical Evidence Summary**
   - Table of all corrected tests
   - Effect size forest plot
   - Convergence scores

7. **Control Comparison**
   - Key: do controls show similar drift?
   - Side-by-side summary
   - Editorial vs. author attribution

8. **Methodology Appendix**
   - Complete description of every feature
   - Statistical test descriptions
   - Parameter choices and justification
   - Software versions and reproducibility info

## 4. CLI Integration

```
uv run forensics report                   # render all notebooks
uv run forensics report --notebook 07     # render specific notebook
uv run forensics report --format html     # HTML only
uv run forensics report --format pdf      # PDF only
```

The `report` command should:
1. Verify `data/analysis/` contains results for all configured authors
2. Execute notebooks via `jupyter nbconvert --execute` or `quarto render`
3. Output to `data/reports/` (rendered HTML/PDF) and `docs/site/` (Quarto site)

## 5. Makefile

Create or update `Makefile` at project root:

```makefile
.PHONY: scrape extract analyze report all deploy clean

scrape:
	uv run forensics scrape

extract:
	uv run forensics extract

analyze:
	uv run forensics analyze

report:
	uv run forensics report

all:
	uv run forensics all

quarto-build:
	cd notebooks && quarto render --to html --output-dir ../docs/site

deploy: quarto-build
	npx wrangler pages deploy docs/site --project-name=ai-writing-forensics

lint:
	uv run ruff check .
	uv run ruff format --check .

test:
	uv run pytest tests/ -v

clean:
	rm -rf data/features/*.parquet
	rm -rf data/embeddings/*.npy
	rm -rf data/analysis/*.json
	rm -rf data/reports/*
	rm -rf docs/site/*
```

## 7. Cloudflare Pages Deployment (Optional)

### CI/CD via GitHub Actions

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy Report
on:
  push:
    branches: [main]
    paths: [notebooks/**, src/**]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          lfs: true
      - uses: quarto-dev/quarto-actions/setup@v2
      - run: quarto render notebooks/ --to html --output-dir docs/site
      - uses: cloudflare/pages-action@v1
        with:
          apiToken: ${{ secrets.CF_API_TOKEN }}
          accountId: ${{ secrets.CF_ACCOUNT_ID }}
          projectName: ai-writing-forensics
          directory: docs/site
```

