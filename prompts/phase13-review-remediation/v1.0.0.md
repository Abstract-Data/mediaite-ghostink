#### Step E2: Switch Simhash to xxhash
**Source:** P3-PERF-001
**Risk:** LOW
**Files:** `src/forensics/utils/hashing.py:L18-45`, `pyproject.toml` (add `xxhash` dependency)
**What:** Replace SHA-256 per n-gram with xxhash for ~10-50x speedup on the fingerprinting step:
```python
import xxhash

def simhash(text: str, hashbits: int = 128) -> int:
    ...
    for gram in grams:
        h = xxhash.xxh128(gram.encode("utf-8")).intdigest()
        for i in range(hashbits):
            if h & (1 << i):
                vector[i] += 1
            else:
                vector[i] -= 1
    ...
```

## Risk Mitigation

- **C1 (_ingest_author_posts):** TDD approach. Write `_ingest_single_post` tests first. Keep old code path behind a flag until new version is verified with integration tests.
- **B1 (ConvergenceInput migration):** 4 call sites change simultaneously. Run full convergence + orchestrator + comparison test suite after each call site update.
- **D1 (HypothesisTest freeze):** Callers already use `tests = apply_correction(tests, ...)` pattern, so the return-value change should be transparent. Verify no code holds stale references.
- **E2 (xxhash):** Changes simhash fingerprint values. Document in HANDOFF.md that existing `dedup_simhash` columns need recalculation. Consider: run dedup recalc as a migration step.
- **E1 (LazyFrame return):** Backward-incompatible for callers expecting DataFrame. The rename-to-eager strategy provides a migration path.
