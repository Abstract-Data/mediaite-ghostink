## Mission

Turn five diagnostic findings from the Apr 2026 run-8 review into a single
landed implementation that (a) restores signal fidelity to the BOCPD,
convergence, drift, and hypothesis-testing stages, (b) excludes shared-byline
accounts from the survey, (c) cuts wall-clock time on large-author
analyses (700+ articles) by an order of magnitude, and (d) enriches every
article with a section-tag derived from the URL path and adds descriptive
+ covariate-adjusted analyses that differentiate articles by tag and
per-author × per-tag aggregates, all while preserving stage boundaries and
the data-model contract.

**Baseline (pre-Phase-15):**

- BOCPD emits `p_cp ≡ hazard_rate` for every feature and every timestep across
  all 10 authors. The detection rule is structurally unable to fire under the
  current Adams & MacKay constant-hazard posterior read.
- Convergence windows require `>= 60%` of 23 raw features to have
  change-points in the window. Effective feature dimensionality after
  collapsing correlated families is ~8–10, so the 60% cutoff asks for
  approximately 80% of independent axes. Four known-interesting windows
  (david-gilmour 2025-12, jennifer-bowers-bahney 2026-03, isaac-schorr
  2025-10, sarah-rumpf 2025-11) fall below threshold despite meaningful
  sub-threshold shifts.
- The top-scoring "author" (mediaite-staff, composite 0.65, 738 articles) is a
  shared byline, not a person. `mediaite` also qualifies. `qualification.py`
  applies no slug/name heuristic.
- Pipeline B returns 0.0 for 7/9 individual authors. Both component signals
  (`peak_signal`, `sim_signal`) floor to zero under noisy monthly data with no
  net drift. It is unclear whether the drift artifacts are also silently
  failing to write for those authors.
- Zero of 2,277 hypothesis tests are FDR-significant (BH q<0.05). BH is
  applied per-author, not globally, so the density issue is intrinsic to
  correcting 200+ tests from a single author across highly correlated
  features.
- `run_full_analysis` processes authors serially. `bootstrap_ci` runs a 200-
  iteration pure-Python loop. Feature-series extraction + median is
  recomputed per change-point rather than per feature. BOCPD and PELT run
  serially across 23 features per author.
- The `metadata` JSON column is populated for only 11 of 77,862 articles
  (0.01%) — a consequence of `bulk_fetch_mode = true`. The article body
  texts carry no inline tag footers. But the URL first-path-segment is
  a reliable section tag at 100% coverage: `media` (73.9%), `politics`
  (19.2%), `opinion` (1.9%), and lower-volume `analysis`, `sponsored`,
  `partner-content`, `crosspost`, `columnists`, `premium`. Every stage
  downstream of extraction currently pools all sections together, which
  (i) lets advertorial/syndicated contaminate stylometric baselines and
  (ii) turns section-mix shift into a latent confound that mimics
  stylometric drift.

### Phase A: BOCPD Detection-Rule Fix (signal correctness, 3–5 hours)

The current `detect_bocpd` thresholds `P(r_t = 0 | x_{1:t})`. Under the
constant-hazard A&M update, this quantity collapses algebraically to the
hazard rate at every timestep, regardless of data — see `log_pi_new[0] =
log_h + log_evidence`, continuation mass `= (1-h) * evidence`, normalization
= `evidence`, therefore `p_cp ≡ h`. No σ² tuning or Student-t prior swap
can move this quantity off `h`.

#### Step A2: Rewrite detection rule to MAP run-length reset
**Risk:** MEDIUM
**Files:** `src/forensics/analysis/changepoint.py` (`detect_bocpd`,
`_bocpd_step`)
**What:** Emit a change-point at timestep `t` when the posterior MAP
run-length drops below a configurable fraction of its previous value, not
when `P(r=0)` exceeds a threshold. The posterior run-length distribution is
already materialized as `log_pi_new`. Full rule with edge cases:

```python
def detect_bocpd(
    signal: np.ndarray,
    hazard_rate: float = 1 / 250.0,
    map_drop_ratio: float = 0.5,
    min_run_length: int = 5,
    reset_cooldown: int = 3,    # refractory: suppress resets for K steps after emit
    merge_window: int = 2,      # multi-reset collapse: merge resets within W steps
) -> list[tuple[int, float]]:
    x = np.asarray(signal, dtype=float).ravel()
    n = len(x)
    if n < max(6, min_run_length + 2):
        return []
    prior = _bocpd_init_prior(x, hazard_rate)
    log_pi = np.array([0.0])
    prev_map = 0
    last_emit_t = -10_000
    raw_emits: list[tuple[int, float]] = []
    for t in range(1, n):
        log_pi, _ = _bocpd_step(t, float(x[t]), log_pi, prior)
        current_map = int(np.argmax(log_pi))
        # Warmup gate: ignore resets until we've built up at least
        # min_run_length of stable history. prev_map must also clear
        # the bar — otherwise the first legitimate reset after a
        # previous reset would re-fire immediately.
        warmed_up = prev_map >= min_run_length
        # Refractory: don't emit within reset_cooldown of the last emit.
        in_cooldown = (t - last_emit_t) < reset_cooldown
        reset_detected = current_map < prev_map * map_drop_ratio
        if warmed_up and reset_detected and not in_cooldown:
            confidence = float(np.exp(log_pi[current_map]))
            raw_emits.append((t, confidence))
            last_emit_t = t
        prev_map = current_map
    # Multi-reset collapse: if the MAP drops across several consecutive
    # timesteps (e.g. 50 → 10 → 2) we want ONE CP, not three. Merge
    # emits within merge_window by keeping the highest-confidence
    # timestamp in each cluster.
    if merge_window <= 0 or len(raw_emits) <= 1:
        return raw_emits
    merged: list[tuple[int, float]] = []
    cluster = [raw_emits[0]]
    for t_i, c_i in raw_emits[1:]:
        if t_i - cluster[-1][0] <= merge_window:
            cluster.append((t_i, c_i))
        else:
            merged.append(max(cluster, key=lambda p: p[1]))
            cluster = [(t_i, c_i)]
    merged.append(max(cluster, key=lambda p: p[1]))
    return merged
```

Confidence returned is the posterior mass at the new MAP run-length
(meaningful, bounded in [0, 1], unlike `P(r=0)` which is pinned to `h`).
Keep `_bocpd_step` unchanged — the update itself is correct A&M; only the
read-out changes.

**Edge cases explicitly handled:**

- **Warmup:** no emits until `prev_map >= min_run_length`.
- **Refractory / cooldown:** `reset_cooldown` suppresses emits within
  K steps of the previous emit. Default 3.
- **Multi-reset collapse:** `merge_window` merges adjacent resets into
  a single CP at the highest-confidence timestep. Default 2.
- **Rollback:** keep the old `P(r=0)` threshold code under a
  `bocpd_detection_mode = "p_r0_legacy"` settings flag for at least
  one release. Default `"map_reset"`. Switching is a one-line config
  change.

**Validation:** `uv run pytest tests/unit/test_bocpd_semantics.py -v`
(flip the xfail to expected-pass in the same commit). Tests must
cover: mean-shift fixture produces exactly one CP (not three from
multi-reset), warmup suppresses early-signal resets, cooldown
suppresses back-to-back emits, rollback flag preserves legacy
behavior byte-for-byte.

#### Step A4: Upgrade the predictive to Student-t (ship directly after A2)
**Risk:** MEDIUM
**Files:** `src/forensics/analysis/changepoint.py` (`_BocpdPrior`,
`_bocpd_init_prior`, `_bocpd_step`)
**What:** With the MAP-reset rule in place, fixed σ² is the binding
constraint on BOCPD sensitivity. Replace the Normal-known-variance
conjugate with Normal-Inverse-Gamma (NIG) → Student-t posterior
predictive. Standard closed-form update; see Murphy 2007 §7.6.

**Promotion rationale (v0.3.0):** v0.1.0/v0.2.0 listed this as optional.
Revised: A2 alone gives us a MAP-reset rule over a miscalibrated
predictive. Without A4, long-established segments still read new
observations through a σ² frozen at the 80-sample prefix — too wide,
so outliers don't look outlying. Student-t is the principled fix and we
are already in the file. Ship A4 in the same sprint as A2.

**Full NIG conjugate update (from Murphy 2007 §7.6, closed form):**

Prior: `(μ_0, κ_0, α_0, β_0)` — default `(mean(x[:10]), 1.0, 1.0,
0.5 * var(x[:10]))`.

Per-observation update for a segment of length `n` with sum `s` and
sum-of-squares `ss`:

```
κ_n = κ_0 + n
μ_n = (κ_0 μ_0 + s) / κ_n
α_n = α_0 + n/2
β_n = β_0 + 0.5 (ss − n (s/n)²) + 0.5 κ_0 n (s/n − μ_0)² / κ_n
```

Posterior predictive at `x_t` under run-length `r` (segment length
`r+1` after observing `x_t`) is a Student-t with:

```
ν    = 2 α_n
loc  = μ_n
scale² = β_n (κ_n + 1) / (α_n κ_n)

log p(x_t | r) = log_studentt_pdf(x_t; ν, loc, scale)
```

Use `scipy.stats.t.logpdf(x_t, df=ν, loc=loc, scale=scale)` — vector-
safe, handles ν → ∞ limit correctly (reduces to Normal).

Prefix-sum bookkeeping: add a `cumsum_sq` array alongside the existing
`cumsum` in `_BocpdPrior` so per-step `ss` is O(1) just like `s`.

Do this as a separate commit so the MAP-reset change is reviewable on its
own. Gate under settings flag `bocpd_student_t: bool = True` (default
on). Fallback to the Normal-known-variance path preserved for A/B.
**Validation:** `uv run pytest tests/ -v` + visual check on the synthetic
mean-shift fixture — Student-t should surface the shift earlier than Normal
with comparable σ².

**Exact `pipeline_a_score` formula under family grouping:**

```python
# Per family, the "representative feature" is the CP with the
# largest product weight * |effect| within that family in the window.
# pipeline_a_score is the unweighted mean of those per-family
# representative effects, clipped to [0, 1].

family_reps: dict[str, float] = {}
for cp in cps_in_window:
    fam = FEATURE_FAMILIES[cp.feature_name]
    score = float(cp.confidence) * abs(cp.effect_size_cohens_d)
    family_reps[fam] = max(family_reps.get(fam, 0.0), score)

if not family_reps:
    return [], 0.0, 0.0
ratio = len(family_reps) / FAMILY_COUNT
pipeline_a_score = float(min(1.0, np.mean(list(family_reps.values()))))
```

and lexical-richness families which happen to be larger. One vote per

representative — noise-dominated weak-effect features do not drag the
family score down.

#### Step C2: Apply BH per feature-family
**Risk:** MEDIUM
**Files:** `src/forensics/analysis/orchestrator.py`
(`_run_hypothesis_tests_for_changepoints`),
`src/forensics/analysis/statistics.py` (`apply_correction`)
**What:** After collecting all tests for one author, group them by
`FEATURE_FAMILIES[test.feature_name]`. Run `apply_correction` once per
family. Concatenate the corrected lists before returning.

Rationale: features within the same family test correlated hypotheses
(e.g. flesch_kincaid / coleman_liau / gunning_fog all read "this author
got easier to read"). BH assumes independence; grouping preserves independence
across families and reclaims statistical power.

Add a module-level note in `statistics.py` explaining why per-family BH is
methodologically preferable to per-author BH for this dataset.

**Edge cases:**

- **Empty family:** if a family has zero tests, skip it (no BH call).
- **Singleton family:** 1 test per family — BH returns raw p unchanged.
  This is correct behaviour but worth a comment in-code so a future
  reader doesn't try to "fix" it.
- **Within-family correlation:** even after per-family grouping,
  `flesch_kincaid` and `coleman_liau` within the `readability` family
  remain correlated. BH still over-corrects in this case. v0.4.0
  scope: document the residual conservatism in the module docstring
  and leave effective-N adjustment as a follow-up. Do NOT attempt
  within-family decorrelation in this phase — the infrastructure
  (correlation matrix estimation per author, effective-N formula) is
  its own research program.

**Validation:** `uv run pytest tests/ -k "orchestrator or statistics" -v`.

_SHARED_TOKENS: frozenset[str] = frozenset({
    "staff", "editors", "newsroom", "team", "desk", "contributor",
    "contributors", "bureau", "wire",
})

def is_shared_byline(
    slug: str,
    display_name: str,
    outlet: str,
) -> bool:
    slug_low = slug.lower()
    name_low = display_name.lower()
    outlet_prefix = outlet.split(".")[0].lower()
    # Outlet-prefixed slug (e.g. `mediaite-staff`, `mediaite`)
    if slug_low == outlet_prefix:
        return True
    if slug_low.startswith(f"{outlet_prefix}-"):
        return True
    # Token match
    if any(tok in slug_low.split("-") for tok in _SHARED_TOKENS):
        return True
    # Multi-author ("Jane Doe and John Smith")
    if " and " in name_low or " & " in name_low:
        return True
    # Comma-separated bylines
    if "," in name_low:
        return True
    return False

### Phase F0: PELT Kernel Swap — RBF → L2 (HIGHEST-LEVERAGE PERF, 30 min)

Profile evidence (Apr 24 2026, isaac-schorr): **99.2% of analysis wall-
clock is spent in `costrbf.error`** (ruptures RBF kernel evaluation) across

**The fix is one line, and it is also the methodologically correct choice.**

#### Step F0.1: Swap `model="rbf"` to `model="l2"` in `detect_pelt`
**Risk:** LOW (semantic change is minor and aligned with Phase C1)
**Files:** `src/forensics/analysis/changepoint.py:55`
**What:**

```python

# After
algo = rpt.Pelt(model="l2", min_size=5).fit(y.reshape(-1, 1))
```

Expose as a settings knob for audit/rollback:

```python

```

Default `l2`. `l1` available for outlier-robust runs. `rbf` stays
available for comparison runs but is no longer the default.

**Why L2 is the right choice here:**

- The forensic question is "did the mean of feature X shift?" — a 1-D
  mean-shift detection problem. L2 cost (sum of squared deviations from
  segment mean) is the textbook-optimal choice for this question;
  the estimator matches the question.
- RBF cost is designed to detect arbitrary distributional shifts (mean,
  variance, higher moments) via a kernel mean embedding. This is
  overkill when we only care about location shifts, and it computes an
  O(n²) pairwise kernel matrix to do so.
- Phase C1 (drop KS) already narrowed the hypothesis-testing battery to
  Welch + Mann–Whitney — both location-shift tests. L2 PELT is
  consistent with that choice; RBF PELT was detecting change-points
  that Welch+MW would then ignore.
- Expected throughput: **100–500× on PELT phase alone.** Tommy
  Christopher's 8,301-article analysis should drop from hours to
  seconds.

**What we give up:** pure-variance-shift / distribution-shape change-
points. In practice these weren't what the reports were reading out; if
we need that signal back later, re-enable under `pelt_cost_model = "rbf"`
per-feature or per-author.

#### Step F1: Vectorize `bootstrap_ci`
**Risk:** LOW
**Files:** `src/forensics/analysis/statistics.py` (`bootstrap_ci`)
**What:** Replace the Python `for _ in range(n_bootstrap)` loop with a
single NumPy call:

```python
def bootstrap_ci(
    group1: list[float],
    group2: list[float],
    n_bootstrap: int = 1000,
    alpha: float = 0.05,
    *,
    seed: int = 42,
) -> tuple[float, float]:
    a = np.asarray(group1, dtype=float).ravel()
    b = np.asarray(group2, dtype=float).ravel()
    if a.size == 0 or b.size == 0:
        return (0.0, 0.0)
    rng = np.random.default_rng(seed)
    s1 = rng.choice(a, size=(n_bootstrap, a.size), replace=True).mean(axis=1)
    s2 = rng.choice(b, size=(n_bootstrap, b.size), replace=True).mean(axis=1)
    diffs = s2 - s1
    lo = float(np.percentile(diffs, 100 * alpha / 2))
    hi = float(np.percentile(diffs, 100 * (1 - alpha / 2)))
    return (lo, hi)
```

#### Step J5: (Conditional) section-adjusted change-point analysis
**Risk:** MEDIUM — gated by J3
**Files:** `src/forensics/analysis/changepoint.py`

```python
import numpy as np

def residualize_by_section(
    y: np.ndarray,
    sections: list[str],
    min_per_section: int = 10,
    condition_number_cap: float = 1e6,
) -> tuple[np.ndarray, dict]:
    """Remove per-section mean from y via OLS on one-hot sections.

    Returns (residuals, diagnostics) where diagnostics reports the
    disposition: "ok", "single_section", "sparse_section",
    "rank_deficient", or "ill_conditioned".
    """
    diag = {"disposition": "ok", "n_sections_used": 0}
    from collections import Counter
    counts = Counter(sections)
    keep = {s for s, n in counts.items() if n >= min_per_section}
    if len(keep) < 2:
        diag["disposition"] = "single_section" if len(keep) == 1 else "sparse_section"
        return y, diag
    # Mask to articles in retained sections; others pass through unchanged.
    mask = np.array([s in keep for s in sections])
    if mask.sum() < len(keep) * min_per_section:
        diag["disposition"] = "sparse_section"
        return y, diag
    sec_list = sorted(keep)
    idx = {s: i for i, s in enumerate(sec_list)}
    X = np.zeros((int(mask.sum()), len(sec_list)))
    for row, s in enumerate(
        [s for s, m in zip(sections, mask, strict=True) if m]
    ):
        X[row, idx[s]] = 1.0
    # Condition number check — degenerate designs pass through.
    if np.linalg.cond(X) > condition_number_cap:
        diag["disposition"] = "ill_conditioned"
        return y, diag
    beta, *_ = np.linalg.lstsq(X, y[mask], rcond=None)
    out = y.copy()
    out[mask] = y[mask] - X @ beta
    diag["disposition"] = "ok"
    diag["n_sections_used"] = len(sec_list)
    return out, diag
```

Degenerate cases (single section, sparse section, rank-deficient
design) return `y` unchanged with a diagnostics entry that surfaces in
the JSON artifact so a reviewer can see which (author × feature) pairs
were residualized vs. passed through.

Gate under `section_residualize_features: bool = False` (off by default
so A/B is a one-line config flip). When enabled, emit BOTH the
unadjusted and adjusted change-point lists in the JSON artifact, with
`method` labels `"pelt"` / `"bocpd"` and
`"pelt_section_adjusted"` / `"bocpd_section_adjusted"`.

**Convergence dispatch (closes gap #5):** which CP set convergence
consumes is controlled by `convergence_cp_source: Literal["raw",
"section_adjusted"] = "section_adjusted"`. Default `"section_adjusted"`
when J5 is enabled — the whole point of residualization is that the
adjusted CPs are the ones we report on. `"raw"` is available for
diagnostic comparison runs. When `section_residualize_features =
False`, only raw CPs exist and `convergence_cp_source` is ignored.

This directly addresses "is the drift stylistic, or section-mix?"
**Validation:** `uv run pytest tests/ -k "residualize or changepoint" -v`.

## Risk Mitigation

- **A2 (BOCPD MAP rewrite):** TDD with the xfail-locked synthetic fixture.
  Do not remove `bocpd_threshold` until A3 ships so rollback is a single
  config toggle.
- **A4 (Student-t upgrade):** Gate under `bocpd_student_t: bool` so A/B
  comparison is trivial. Keep the Normal path alive for one release.
- **B3 (threshold drop 0.60 → 0.50):** Semantic shift. Bump the analysis
  config-hash so old artifacts don't silently mix with new ones.
- **C2 (per-family BH):** Keep a `fdr_grouping: "author" | "family"` config
  so the old behaviour is selectable for replication/auditing.
- **D3 (shared-byline filter):** Default-exclude but expose `--include-
  shared-bylines` on the CLI. Dump a CSV of excluded slugs into
  `data/survey/excluded_shared.csv` for transparency.
- **E3 (signal recalibration):** Only ship if E1 confirms math-floor. Keep
  old formulas behind a `pipeline_b_mode: "legacy" | "percentile"` knob.
- **G1 (process-pool parallelism):** Each worker opens its own Repository;
  do NOT share a connection. Test H2 (byte-identical artifacts) is the
  hard gate.
- **G2 (feature-level parallelism):** Ship disabled (`feature_workers=1`).
  Only flip on after big-N benchmarks prove per-author parallelism isn't
  already saturating.
- **J2 (advertorial exclusion):** Default-exclude but expose
  `--include-advertorial`. Dump excluded rows to CSV so the editorial
  decision is auditable. Some authors may lose so many articles that
  they drop below the `min_articles` qualification bar — this is
  correct behaviour (they weren't writing enough organic articles to
  qualify), but note it in HANDOFF.md.
- **J5 (section-adjusted change-points):** Gated behind the J3
  diagnostic AND the `section_residualize_features` config flag. Emit
  BOTH adjusted and unadjusted CPs in the JSON so reviewers can see the
  delta. OLS on one-hot sections is numerically trivial but degenerates
  when only one section has data — the helper returns `y` unchanged in
  that case. Add an explicit test for the degenerate path.
- **J6 (section contrast tests):** Multiply tests by author × section-
  pair × feature. Apply per-family BH (same helper as C2) to keep the
  FDR denominator honest. Skip authors with < 2 sections × 30 articles
  cleanly rather than emitting low-power results.
- **F0 (PELT L2 swap):** Semantic change from full-distribution to
  mean-shift detection. Aligned with Phase C1's KS drop — the whole
  analytical stack now focuses on location shifts consistently. Keep
  `pelt_cost_model` as a settings knob so `rbf` is selectable for
  comparison runs. If the snapshot diff loses > 20% of high-|d| RBF
  CPs, pause and investigate. The rerun-on-all-authors step in the
  fast-path workflow is the empirical go/no-go gate.
- **Phase 0 as hard blocker:** If Phase 0 is skipped or landed
  out of order, expect to re-touch `settings.py` 5+ times and create
  rebase conflicts on every subsequent phase. The 2–3 hour cost is
  amortized across the remaining 9 phases.
- **Phase L pre-conditions:** L1, L3, and L5 are moral pre-conditions,
  not technical ones — the code works without them. But the
  forensic-defensibility of the final report requires them. If you
  can't get maintainer sign-off on the pre-registration amendment
  (L3) quickly, land L1 + L5 and mark L3 as in-flight in HANDOFF.md.
- **Phase K (reporting integration):** UI-adjacent work has a history
  of needing revision passes. Plan for one review cycle with the
  project maintainer before declaring K complete. Charts should
  render correctly in the Cloudflare-deployed report (if
  `cloudflare_deploy = true` in config), not just locally.
- **Phase 0 Step 0.3 (parquet migration):** the most complex single
  step. Back up `data/features/` before running the migration script.
  Test on a copy first. Keep the `_pre_phase15_backup/` directory on
  disk until Phase 15 lands and reports validate.
- **A2 edge cases (cooldown, warmup, merge):** the three new knobs
  (`bocpd_reset_cooldown`, `bocpd_min_run_length`,
  `bocpd_merge_window`) multiply the test surface. Reference-fixture
  tests in H1 must cover each knob independently and in combination.
