## Mission

Implement every actionable finding from the Phase A / Phase B adversarial
architecture review (run on 2026-04-25 using the project-adapted
`adversarial-review` skill). The Phase A pass produced 26 ranked risks
across the eight pipeline attack surfaces; the Phase B Builder pass
returned **17 accepts** and **9 rejects-with-technical-rebuttal**.

This prompt is the implementation contract for the 17 accepts. It is
self-contained: a future agent should be able to execute the entire
remediation from this file alone, without consulting the original Phase A
or Phase B output.

The work materially strengthens four specific claims made in
`mediaite-ghostink-integrity-report.docx` that the adversarial pass found
were not fully backed by the code:

1. The encoder is "pinned to a specific revision" — currently false.
2. "No individual setting can drift without a visible mismatch" — currently
   false for `pelt_penalty`, `bocpd_hazard_rate`, `bocpd_min_run_length`,
   and `min_articles_for_period`.
3. The corpus hash gives an auditor "a cryptographic anchor to verify two
   runs hashed against the same corpus" — currently order-dependent on
   article UUIDs, so re-scrapes of identical corpora hash differently.
4. Every confirmatory finding is bound to its frozen configuration —
   currently undermined by silent NaN-propagation through Welch's t,
   Mann–Whitney, and Benjamini–Hochberg.

After Phase 16 lands, every credibility claim in the integrity report
will be backed by enforceable code. Phases 16 J and 16 I additionally
close a set of lower-priority defensive gaps and amend the integrity
report's scope where it over-claimed.

**Baseline (pre-Phase-16):**

- `features/embeddings.py:_get_model()` calls `SentenceTransformer(model_name)`
  without `revision=`. The HuggingFace alias resolves to whatever revision
  is current at load time. `EmbeddingRecord.model_version` is a free-form
  string ("v2.0") decoupled from the actual checkpoint.
- `AnalysisConfig` flags only a subset of signal-bearing fields with
  `json_schema_extra={"include_in_config_hash": True}`. Knobs that affect
  PELT/BOCPD/sample-size gating (`pelt_penalty`, `bocpd_hazard_rate`,
  `bocpd_min_run_length`, `min_articles_for_period`) are missing the marker
  and can change without invalidating any locked artifact.
- `compute_corpus_hash()` does `SELECT content_hash FROM articles
  ORDER BY id`. `Article.id` defaults to `str(uuid4())`. Two runs over the
  identical corpus produce different hashes if insertion order differs.
- `run_hypothesis_tests()` returns `[]` (no log) for `n < 4` or imbalanced
  splits; `mannwhitneyu` falls back to `p_mw = 1.0` on ValueError;
  `cohens_d` returns `0.0` on zero pooled-std. NaN feature values
  propagate through `scipy.stats.ttest_ind` and poison
  `apply_correction()` — the entire family's adjusted p-values become NaN
  and silently compare as non-significant.
- `Repository._connect()` sets `check_same_thread=False` and the docstring
  states callers must serialize all reads/writes externally — but no
  internal lock enforces this contract.
- Several config fields lack `Field(ge=...)` validators: `word_count`,
  `bocpd_hazard_rate`, `min_articles_for_period`. SQLite has no
  `CHECK (word_count >= 0)` constraint. `changepoint_methods: list[str]`
  accepts arbitrary strings.

**Scope:** Implement Phases A–K below in order. No stage-boundary
rewrites. No data-model contract breaks except where explicitly noted
(new `embedding_model_revision` field on `AnalysisConfig`, new
`model_revision` field on `EmbeddingRecord`, new `skipped_reason`,
`degenerate`, `n_pre`, `n_post` fields on `HypothesisTest`, new
`schema_version` field on `corpus_custody.json`, `CHECK (word_count >= 0)`
migration on `articles`).

```python
def _get_model(model_name: str, revision: str) -> Any:
    cache_key = f"{model_name}@{revision}"

    def load() -> Any:
        from sentence_transformers import SentenceTransformer

        logger.info(
            "Loading sentence-transformer model: %s @ %s", model_name, revision
        )
        return SentenceTransformer(model_name, revision=revision)

    return _ST_MODEL_CACHE.get_or_load(cache_key, load)


def compute_embedding(
    text: str,
    model_name: str,
    revision: str,
) -> np.ndarray:
    model = _get_model(model_name, revision)
    vec = model.encode(text or "", show_progress_bar=False)
    return np.asarray(vec, dtype=np.float32)
```

Note the cache key change: `model_name + revision` rather than

```python
def run_hypothesis_tests(
    feature_values: list[float],
    breakpoint_idx: int,
    feature_name: str,
    author_id: str,
    *,
    n_bootstrap: int = 1000,
    enable_ks_test: bool = False,
) -> list[HypothesisTest]:
    raw = list(feature_values)
    n_raw = len(raw)
    if n_raw == 0 or breakpoint_idx < 1 or breakpoint_idx >= n_raw:
        return [_skipped_test(
            feature_name, author_id, "no_data_or_invalid_split",
            n_pre=0, n_post=0, n_nan_dropped=0,
        )]

    # Split first, then drop NaN within each segment so we know which
    # segment lost data.
    pre_raw = raw[:breakpoint_idx]
    post_raw = raw[breakpoint_idx:]
    pre = [v for v in pre_raw if not _is_nan(v)]
    post = [v for v in post_raw if not _is_nan(v)]
    n_dropped = (len(pre_raw) - len(pre)) + (len(post_raw) - len(post))
    if n_dropped > 0:
        logger.warning(
            "NaN values dropped from hypothesis test for %s/%s: "
            "%d pre, %d post (%d total)",
            author_id, feature_name,
            len(pre_raw) - len(pre),
            len(post_raw) - len(post),
            n_dropped,
        )

    if len(pre) < 2 or len(post) < 2:
        return [_skipped_test(
            feature_name, author_id, "insufficient_non_nan",
            n_pre=len(pre), n_post=len(post), n_nan_dropped=n_dropped,
        )]
    # ... existing test logic, now using cleaned pre/post ...
```

#### Step D4: Update apply_correction to exclude skipped+degenerate from BH
**Risk:** MEDIUM
**Files:** `src/forensics/analysis/statistics.py`
**What:** `apply_correction()` and `apply_correction_grouped()` must
partition tests into "rankable" (raw_p_value is finite, skipped_reason
is None, degenerate is False) and "non-rankable" (everything else):

```python
def apply_correction(tests, method="benjamini_hochberg", alpha=0.05):
    rankable = [t for t in tests if t.skipped_reason is None
                and not t.degenerate
                and not math.isnan(t.raw_p_value)]
    non_rankable = [t for t in tests if t not in rankable]

    # Run BH on rankable only; denominator = len(rankable)
    corrected = _apply_bh(rankable, alpha) if rankable else []

    # Non-rankable tests pass through with corrected_p_value = NaN
    for t in non_rankable:
        t = t.model_copy(update={
            "corrected_p_value": float("nan"),
            "is_significant": False,
        })
        corrected.append(t)
    return corrected
```

The denominator change is the meaningful behavior shift: a family of
50 features where 30 are skipped now has `family_size = 20` for BH,
not 50. This restores nominal FDR control.

```python
def compute_rolling_stats(timestamps, values, windows=None):
    wdef = windows if windows is not None else [30, 90]
    if not timestamps or not values:
        return {w: {"mean": [], "std": [], "low95": [], "high95": []} for w in wdef}

    df = pl.DataFrame({"timestamp": timestamps, "value": values}).sort("timestamp")
    expressions = []
    for w in wdef:
        if w < 2:
            continue
        expressions.extend([
            pl.col("value").rolling_mean(window_size=w, min_samples=1).alias(f"mean_{w}"),
            pl.col("value").rolling_std(window_size=w, min_samples=1).alias(f"std_{w}"),
        ])
    rolled = df.with_columns(expressions)
    out: dict[int, dict[str, Any]] = {}
    for w in wdef:
        if w < 2:
            continue
        mean = rolled[f"mean_{w}"].to_list()
        std = [float(s) if s is not None else 0.0 for s in rolled[f"std_{w}"].to_list()]
        out[w] = {
            "mean": [float(x) if x is not None else float("nan") for x in mean],
            "std": std,
            "low95": [float(m - 1.96 * s) for m, s in zip(mean, std, strict=False)],
            "high95": [float(m + 1.96 * s) for m, s in zip(mean, std, strict=False)],
        }
    return out
```

## Out of Scope (Builder Rebuttals)

The following items were raised by Phase A but rejected by Phase B with a
specific technical argument. Recorded here so future agents do not
re-litigate them.

### R6 — "Per-family BH inflates FDR under positive within-family correlation"
**REJECT.** Phase A had the direction wrong. Benjamini–Hochberg is valid
under Positive Regression Dependence on the Subset (PRDS), which
positively correlated readability metrics satisfy. Under positive
correlation BH is *conservative* (lower power, FDR ≤ α), not
anti-conservative (higher FDR). The integrity report's existing wording
("conservative under positive within-family correlation") is correct.

### R7 — "Polars `.sort()` is lazy; PELT/BOCPD may receive unsorted rows"
**PARTIALLY ACCEPTED.** The cited code uses `pl.DataFrame({...}).sort(...)`
— that's an eager DataFrame, not LazyFrame. The lazy concern does not
apply. The legitimate adjacent concern (input validation at
`chow_test`/`stl_decompose`) is addressed in Phase J.4 as defense in
depth.

### R12 — "Cross-stage import in cli/extract.py"
**REJECT.** The CLI orchestration layer is allowed to import from any
stage; the forbidden pattern is *stage modules importing from sibling
stage modules*. `features/pipeline.py` does not import from `scraper/`.
Confirm by `rg "from forensics\.(scraper|analysis|reporting)" src/forensics/(scraper|features|analysis|reporting)/`.

### R14 — "DateTime stored as TEXT, not DATETIME"
**REJECT.** SQLite uses type affinity; `DATETIME` columns are stored as
TEXT internally regardless. The `parse_datetime()` round-trip is the
canonical pattern. SQLite's `date()` and `datetime()` SQL functions
work directly on ISO-8601 TEXT. The actual orthogonal concern
(timezone handling) is documented in Phase J as a sorted-input
assertion and in the integrity report's amendment.

### R16 — "Empty-text simhash collapses all empty articles to one component"
**REJECT.** Empty articles should be excluded; the simhash collision
achieves the right outcome. Phase J.5 adds an explicit empty-content
filter before dedup so the audit trail records "dropped: empty content"
instead of "dropped: simhash collision," which is a cleaner provenance
trail without changing behavior.

### R17 — "apply_correction_grouped iteration ordering"
**REJECT.** CPython 3.7+ guarantees dict insertion-order iteration;
`defaultdict` follows the same rule. Phase A flagged a possible
footgun, not an actual one. The integrity report's determinism claim
is independently tested by `tests/integration/test_parallel_parity.py`.

### R24 — "model_validator(mode='before') on FeatureVector"
**REJECT.** `mode="before"` runs against the input dict, not against
partial field state. The validator is correct for legacy compatibility.
Speculative concerns about future validators are not a current risk.

### R25 — "is_duplicate INTEGER instead of BOOLEAN"
**REJECT.** SQLite has no native BOOLEAN. `INTEGER 0/1` is canonical.
Adding a CHECK constraint is cosmetic; adapters already enforce it.

### R26 — "TUI FIXMEs"
**REJECT.** Out of forensic-credibility scope.
