# Phase 2: Scraper — Author Discovery & Article Metadata

Version: 0.1.0
Status: pending
Last Updated: 2026-04-20
Model: gpt-5-3-codex
Depends on: Phase 1 (scaffolding, models, storage, config)

## Objective

Implement the first step of the two-step scraping approach: use the WordPress REST API to discover all authors at Mediaite, then collect article metadata (URL, title, date) for each configured author. Store the author manifest to `data/authors_manifest.jsonl` and article metadata to SQLite. After this phase, `uv run forensics scrape --discover` populates the manifest, and `uv run forensics scrape --metadata` populates article URLs in the database.

## Target Site

Mediaite (`mediaite.com`) — WordPress-based. The REST API exposes metadata but suppresses article body content (content/excerpt fields return empty strings due to Mediaite+ subscription model). This phase only uses the API for metadata; HTML fetching is Phase 3.

## 1. Author Discovery (src/forensics/scraper/crawler.py)

### discover_authors()

Paginate through the WordPress Users endpoint:

```
GET https://www.mediaite.com/wp-json/wp/v2/users?per_page=100&page={n}
```

Response headers `X-WP-Total` and `X-WP-TotalPages` provide pagination info.

For each user, extract and store an `AuthorManifest`:
- `wp_id`: user["id"]
- `name`: user["name"]
- `slug`: user["slug"]
- `total_posts`: will be populated per-author in the next step
- `discovered_at`: datetime.utcnow()

To get `total_posts` for each author, make a HEAD or GET request to:

```
GET https://www.mediaite.com/wp-json/wp/v2/posts?author={wp_id}&per_page=1&_fields=id
```

Read the `X-WP-Total` header for the count.

Write the full author list to `data/authors_manifest.jsonl` (one JSON line per author). This is a one-time discovery step — if the file already exists, ask whether to refresh or skip.

### Implementation details:
- Use `httpx.AsyncClient` for all requests
- Respect `ScrapingConfig.rate_limit_seconds` with jitter between requests
- Set `User-Agent` from config
- Handle HTTP errors: log and continue on 4xx/5xx for individual authors
- Log total authors discovered and top authors by post count

## 2. Article Metadata Collection (src/forensics/scraper/crawler.py)

### collect_article_metadata(author_slugs: list[str])

For each configured author (from `config.toml` `[[authors]]` entries), fetch all their article metadata via:

```
GET https://www.mediaite.com/wp-json/wp/v2/posts?author={wp_id}&per_page=100&page={n}&_fields=id,slug,link,title,date
```

Paginate using `X-WP-TotalPages`. For each article, create an `Article` model with:
- `id`: generate UUID
- `author_id`: the Author's UUID (match by slug from config)
- `url`: post["link"]
- `title`: post["title"]["rendered"]
- `published_date`: parse post["date"] (ISO 8601)
- `clean_text`: "" (empty — HTML fetch is Phase 3)
- `word_count`: 0
- `content_hash`: ""

Before inserting, check if the URL already exists in SQLite (idempotent/resumable).

### Author matching flow:
1. Load `authors_manifest.jsonl` to get `wp_id` ↔ `slug` mapping
2. For each author in config, find their `wp_id` from the manifest
3. If an author slug from config isn't found in the manifest, log a warning and skip

### Store results:
- Upsert the `Author` record into SQLite (from config + manifest data)
- Upsert each `Article` record into SQLite (metadata only, no body yet)
- Log: "{N} articles indexed for {author_name} ({date_range})"

## 4. Rate Limiting & Resilience (src/forensics/scraper/fetcher.py)

```python
class RateLimiter:
    """Async rate limiter with jitter."""

    def __init__(self, delay: float, jitter: float):
        self.delay = delay
        self.jitter = jitter
        self._last_request = 0.0

    async def wait(self):
        """Wait until it's safe to make the next request."""
        ...
```

Also implement retry logic:
- Retry up to `max_retries` times on 5xx or connection errors
- Exponential backoff: `retry_backoff_seconds * 2^attempt`
- On 429 (rate limited): respect `Retry-After` header if present, else back off 30s
- On 404: log and skip (dead endpoint)
- Log all errors to `data/scrape_errors.jsonl`
